# Тема урока:

- AddSingleton, AddScoped, AddTransient

## AddSingleton, AddScoped, AddTransient

`AddSingleton` - это метод добавления вашего
сервиса в контейнер зависимостей, который будет создаваться один раз за всё время жизни приложения. У таких объектов есть проблема с
многопоточностью, так как они являются общими для всех потоков. Многие избегают использования данного подхода, но у него есть и плюсы. Например, если вы сами создаете Middleware, то вам нужно использовать `AddSingleton`, так как Middleware создается один раз и используется для всех запросов. Или же мы можем взять в пример `EmailSenderService`, который будет создаваться один раз и использоваться для всех запросов. Он используется в приложение достаочно редко, поэтому нет смысла создавать его каждый раз заново.

`AddScoped` - это метод добавления вашего сервиса в контейнер зависимостей, который будет создаваться один раз за один запрос. Это значит, что если у вас есть 10 запросов, то ваш сервис будет создан 10 раз. Это хороший подход, так как у вас не будет проблем с многопоточностью, так как каждый запрос обрабатывается в своем потоке. Например, если у вас есть сервис, который работает с базой данных, то вам нужно использовать `AddScoped`, так как у вас не будет проблем с многопоточностью. Я для себя вывел правило, что если у вас есть сервис, который работает с базой данных, то используйте `AddScoped`.

`AddTransient` - это метод добавления вашего сервиса в контейнер зависимостей, который будет создаваться каждый раз, когда кто-то обратится к нему. Данный метод создает новый объект каждый раз, когда кто-то обращается к нему. Даже внутри одного
запроса он может пересоздаваться несколько раз. Его используют в случаях, когда у вас есть сервис, который не хранит состояние. Напрмиер, для каких-то легковесных сервисов
которые должны что-то сделать не храня при этом состояние чего-либо.
