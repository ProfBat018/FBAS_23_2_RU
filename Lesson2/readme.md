## 1. Анализ типов проектов для Microsoft Azure
В Microsoft Azure существует несколько типов проектов, которые можно создавать и развертывать. Основные из них:
- **Web Apps**: Позволяют разрабатывать и размещать веб-приложения на платформе Azure. Они поддерживают различные языки программирования, такие как .NET, PHP, Node.js и Python.

- **Cloud Services**: Позволяют создавать и развертывать облачные приложения, которые могут состоять из нескольких компонентов, таких как веб-приложения, службы и базы данных.

- **Virtual Machines**: Позволяют создавать и управлять виртуальными машинами в облаке Azure. Это позволяет запускать любые приложения и операционные системы, которые поддерживаются на виртуальных машинах.

- **Functions**: Позволяют создавать серверлесс-приложения, которые выполняются в ответ на события. Это позволяет экономить ресурсы и платить только за фактическое использование.

- **Containers**: Позволяют разрабатывать и развертывать контейнеризованные приложения с использованием Docker и Kubernetes. Это позволяет легко управлять зависимостями и масштабировать приложения.

#### Web Apps
Web Apps - это любые веб приложения, начиная с обычного веб-сайта и заканчивая самым прогрессивным веб-приложением. Одним из плюсов Azure является то, что он поддерживает в принципе все языки программирования, на которых можно писать веб-приложения.

#### Cloud Services
Данный тип проектов позволяет создавать и развертывать облачные приложения, которые могут состоять из нескольких компонентов, таких как веб-приложения, службы и базы данных. Это позволяет создавать сложные приложения с высокой доступностью и масштабируемостью.
По факту это более комплексное решение, которое может состоять из нескольких приложений, которые могут взаимодействовать друг с другом. Например, у вас может быть веб-приложение, которое использует облачную службу для обработки данных и базу данных для хранения информации.

#### Virtual Machines
Как я говорил на прошлом уроке, данный `IAAS` сервис позволяет вам по сути арендовать свой компьютер в облаке. Вы можете установить на него любую операционную систему и любое программное обеспечение, которое вам нужно. Это позволяет запускать любые приложения и операционные системы, которые поддерживаются на виртуальных машинах. Абсолютно никаких ограничкений, просто вручную конечно же все сложнее настраивать. 

#### Functions
Если вы помните, то когда мы с вами писали свой Auth сервис, то я говорил вам про такую вещь как `Black List` в котором будут хранится токены, которые мы будем удалять если пользователь вышел из системы. Нашей проблемой было то, что после удаления токена нам нужно было бы как очищать наш список спустя определенное время. Данное решение от Azure позволяет нам сделать хостинг обычной функции, которая либо будет как триггер, либо будет работать по расписанию. 

#### Containers
Вы еще не знаете что такое контейнеры, Docker, Kubernetes и т.д. Но в будущем вы с этим столкнетесь. Azure позволяет вам разрабатывать и развертывать контейнеризованные приложения с использованием Docker и Kubernetes. Это позволяет легко управлять зависимостями и масштабировать приложения.
