# Kubernetes 

Мы уже с вами пользовались Docker-Compose и знаем зачем он нужен. В контексте работы с микросервисной архитектурой, где каждый сервис представляет собой отдельный контейнер, Docker-Compose становится неудобным инструментом. Вместо него используют Kubernetes. По факту такие оркестраторы как `Kubernetes`, `Docker Swarm`, `Apache Mesos` и т.д. позволяют управлять контейнерами, но Kubernetes является самым популярным и широко используемым. `Swarm` уже в принципе устарел и не используется в продакшене.

## Что такое Kubernetes ?
Это оркестратор, который является продуктом компании Google. Он позволяет управлять контейнерами, автомасштабировать их, управлять сетью и т.д. Kubernetes позволяет управлять контейнерами в любом окружении, будь то облако или локальный сервер. 

## Как работает Kubernetes ?

Kubernetes управляет контейнерами в кластере. Кластер - это набор серверов, которые управляются одним мастер-сервером. Мастер-сервер управляет всеми нодами (серверами) в кластере. На каждой ноде устанавливается агент, который управляется мастер-сервером.

## Какие компоненты входят в Kubernetes ?

1. **Master-сервер** - это главный сервер, который управляет всеми нодами в кластере. Он состоит из следующих компонентов:
    - **API Server** - это сервер, который управляет всеми запросами к кластеру. Все запросы к кластеру проходят через API Server.
    - **Scheduler** - это компонент, который отвечает за распределение контейнеров по нодам.
    - **Controller Manager** - это компонент, который отвечает за контроль над всеми контроллерами в кластере.
    - **etcd** - это key-value хранилище, которое хранит все данные о кластере.

2. **Node** - это сервер, на котором запускаются контейнеры. Каждый сервер в кластере называется нодой. Нода состоит из следующих компонентов:
    - **Kubelet** - это агент, который устанавливается на каждой ноде и управляется мастер-сервером.
    - **Kube-proxy** - это компонент, который отвечает за сеть в кластере.
    - **Container Runtime** - это компонент, который отвечает за запуск контейнера.

Так же есть плюшки в виде Kubernetes secret, ConfigMap, Persistent Volume и т.д.

## Как установить Kubernetes ?

Kubernetes можно установить на локальный сервер или на облако. Для локального сервера можно использовать `Minikube`, для облака можно использовать `Google Kubernetes Engine (GKE)`, `Amazon Elastic Kubernetes Service (EKS)`, `Azure Kubernetes Service (AKS)` и т.д.

У нас есть 5 .csproj файла, которые нам надо запустить.
1. **ApiGateway** - это наш API Gateway, который будет принимать все запросы и перенаправлять их на нужный сервис.
2. **AuthService** - это сервис, который отвечает за авторизацию пользователей.
3. **MovieService** - это сервис, который отвечает за работу с фильмами.
4. **CommunicationService** - это сервис, который отвечает за работу с комментариями.
5. **NotificationService** - это сервис, который отвечает за отправку уведомлений.

Для того чтобы запустить наши сервисы в Kubernetes, нам надо создать Docker образы для каждого сервиса и загрузить их в Docker Hub. После этого нам надо создать Kubernetes манифесты для каждого сервиса и запустить их в кластере.
