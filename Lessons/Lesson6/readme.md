## Зачем нужны сети в Docker 

Как вы уже знаете с прошлых уроков, Docker позволяет запускать наши контейнеры в изолированной, я бы даже сказал инкапсулированной среде. Для того чтобы наши контейнеры могли взаимодействовать друг с другом по сети или наоборот, если нам нужно ограничить доступ к контейнеру, нам нужно использовать сети. В целом вы должны принять тот факт что при использовании `Docker` под целое приложение, вам по-любому придется использовать `compose` подход. Единичные случаи использования обычного `docker run` это какие-то системы мониторинга или тестирования. Хотя даже их я поднимаю с `compose`. 

## Типы сетей 
Docker предоставляет несколько типов сетей, которые можно использовать для связи между контейнерами. Вот основные типы сетей:
1. **Bridge**: Это сеть по умолчанию, которая создается при установке Docker. Контейнеры, подключенные к этой сети, могут общаться друг с другом, но не могут обращаться к контейнерам в других сетях.
2. **Host**: Контейнеры используют сетевой стек хоста. Это означает, что контейнеры могут обращаться к сервисам на хосте и наоборот. Однако это также означает, что контейнеры не изолированы друг от друга.
3. **Overlay**: Эта сеть позволяет контейнерам, работающим на разных хостах, общаться друг с другом. Это полезно для кластеров Docker Swarm или Kubernetes.

Если вы запускаете обычный контейнер без compose, то по умолчанию он будет подключен к сети `bridge`. Если вы хотите подключить контейнер к другой сети, вы можете использовать флаг `--network` при запуске контейнера. Например:
```bash
docker run -d --name my_container --network my_network my_image
```

Предварительно вы можете для своего контейнера создать сеть с помощью команды:
```bash
docker network create my_network
```
здесь `my_network` - это имя вашей сети, а `my_container` - это имя вашего контейнера. Для того чтобы указать тип сети, вы можете использовать флаг `--driver`. Например:
```bash 
docker network create --driver bridge my_network
```

`Host` сеть можно использовать если вам нужно чтобы общение проходило между вашей хостовой системой и контейнером. Например, если вы хотите запустить веб-сервер в контейнере и сделать его доступным на порту 80 вашего хоста, вы можете использовать следующую команду:
```bash
docker run -d --name my_container --network host my_image
```
В этом случае контейнер будет использовать сетевой стек вашего хоста, и вы сможете получить доступ к веб-серверу по адресу `http://localhost:80`. 

`Overlay` сеть используется для связи между контейнерами, работающими на разных хостах. Это полезно для кластеров Docker Swarm или Kubernetes. Это мы с вами рассмотрим на следующем уроке, когюа будем делать микросервисы с помощью `Kubernetes`.

## Port forwarding
при создании контейнера, вамм нужно связать вашу хостовую сеть(сеть вашего компьютера) с сетью контейнера. Для этого вам нужно использовать флаг `-p`. Например:
```bash
docker run -d --name my_container -p 8080:80 my_image
```
В этом случае порт 8080 на вашем хосте будет перенаправлен на порт 80 в контейнере. Это означает, что вы сможете получить доступ к веб-серверу в контейнере по адресу `http://localhost:8080`.
