# AZ-900: Chapter 2.1 - Describe the core architectural components of Azure

## Regions

**Что это:**
- Географически распределённые области, где находятся дата-центры Microsoft Azure.
- Каждый регион состоит из одного или нескольких дата-центров, объединённых в сеть с низкой задержкой.

**Примеры:**
- East US, West Europe, Southeast Asia.

**Зачем нужно:**
- Возможность размещения ресурсов ближе к пользователям для повышения производительности и соблюдения требований законодательства о хранении данных.

---

## Region Pairs

**Что это:**
- Связанные пары регионов внутри одного географического региона для обеспечения высокой доступности и восстановления после катастроф.

**Особенности:**
- При возникновении аварии в одном регионе другой регион автоматически принимает нагрузку.
- Обновления Azure проводятся поочередно между парами регионов.

**Пример:**
- West Europe и North Europe образуют пару регионов.

---

## Availability Zones

**Что это:**
- Физически отдельные локации внутри региона с независимым питанием, охлаждением и сетями.
- Минимизируют риск одновременного выхода из строя всех зон.

**Использование:**
- Размещение реплик приложений и данных в разных зонах для обеспечения высокой доступности.

**Пример:**
- В регионе East US 2 есть несколько Availability Zones.

---

## Resource Groups

**Что это:**
- Логические контейнеры для управления ресурсами Azure.
- Позволяют группировать ресурсы, которые имеют общий жизненный цикл.

**Преимущества:**
- Упрощённое управление.
- Совместное применение политик и прав доступа.

**Пример:**
- В одном Resource Group можно хранить виртуальные машины, базы данных и сетевые ресурсы для одного проекта.

---

## Subscriptions

**Что это:**
- Способ организации и управления доступом к ресурсам Azure, а также их оплаты.
- Каждая подписка имеет уникальный ID и привязана к счёту оплаты.

**Преимущества:**
- Разделение ресурсов по проектам, отделам или типам услуг.
- Управление лимитами и политиками безопасности на уровне подписки.

---

## Management Groups

**Что это:**
- Иерархическая структура для организации нескольких подписок Azure.
- Позволяет централизованно применять политики безопасности, управление доступом и соблюдение стандартов.

**Применение:**
- Для крупных компаний с множеством подписок и команд.

**Пример:**
- Создание Management Group для всех подписок финансового отдела.

---

## Azure Resource Manager (ARM)

**Что это:**
- Сервис управления ресурсами Azure.
- Обеспечивает развертывание, управление и контроль за ресурсами через единый API.

**Возможности:**
- Декларативные шаблоны (ARM templates) для автоматизации развертывания.
- Управление доступом (RBAC).
- Применение тегов (tags) для организации ресурсов.

**Пример:**
- Развертывание набора ресурсов через ARM Template для создания инфраструктуры веб-приложения.

---

## Краткий вывод

> Архитектурные компоненты Azure обеспечивают масштабируемость, высокую доступность, удобство управления и безопасность инфраструктуры облака.

# AZ-900: Chapter 2.2 - Describe core Azure services and products

## Compute services

### Virtual Machines (VMs)
**Что это:**
- Эмулированные компьютеры, работающие в дата-центрах Azure.
- Позволяют запускать различные операционные системы и приложения.

**Особенности:**
- Полный контроль над ОС.
- Гибкость в настройке ресурсов (CPU, RAM, диск).
- Возможность масштабирования вручную или автоматически.

**Применение:**
- Размещение веб-серверов, баз данных, приложений.

---

### App Services
**Что это:**
- Платформа для размещения веб-приложений, API и мобильных back-end без необходимости управления инфраструктурой.

**Особенности:**
- Автоматическое масштабирование.
- Поддержка различных языков программирования (Node.js, .NET, Java, Python).
- Интеграция с DevOps процессами.

**Применение:**
- Быстрое развертывание корпоративных сайтов и API.

---

### Azure Kubernetes Service (AKS)
**Что это:**
- Управляемый сервис для автоматизированного развертывания, масштабирования и управления контейнерами Kubernetes.

**Особенности:**
- Упрощённая настройка кластеров Kubernetes.
- Интеграция с мониторингом и безопасностью Azure.

**Применение:**
- Микросервисные архитектуры.
- Высоконагруженные распределённые приложения.

---

### Azure Functions
**Что это:**
- Serverless-сервис для запуска кода по событию без управления серверами.

**Особенности:**
- Платишь только за фактическое выполнение кода.
- Поддержка множества языков.

**Применение:**
- Автоматизация задач.
- Реакция на события (например, обработка сообщений в очередях).

---

## Networking services

### Virtual Network (VNet)
**Что это:**
- Частная сеть в Azure, аналогичная традиционной локальной сети.

**Особенности:**
- Безопасное соединение ресурсов Azure.
- Возможность подключения к локальным сетям через VPN или ExpressRoute.

---

### Load Balancer
**Что это:**
- Сервис для распределения входящего трафика между несколькими ресурсами.

**Особенности:**
- Поддержка внутреннего и внешнего балансировщика.
- Высокая доступность приложений.

---

### VPN Gateway
**Что это:**
- Сервис для создания защищенного соединения между Azure и локальными сетями.

**Особенности:**
- Поддержка различных типов VPN (Point-to-Site, Site-to-Site).

---

### Azure DNS
**Что это:**
- Управляемый сервис для размещения доменных записей DNS в Azure.

**Особенности:**
- Надежное разрешение имён с низкой задержкой.

---

### Content Delivery Network (CDN)
**Что это:**
- Сеть серверов, расположенных по всему миру, для быстрой доставки контента пользователям.

**Особенности:**
- Снижение времени загрузки сайтов.
- Оптимизация пропускной способности.

---

### ExpressRoute
**Что это:**
- Выделенное частное соединение между локальной инфраструктурой и Azure.

**Особенности:**
- Более высокая производительность.
- Более высокая безопасность по сравнению с публичным интернетом.

---

## Storage services

### Blob Storage
**Что это:**
- Сервис для хранения неструктурированных данных (например, изображений, видео, резервных копий).

**Особенности:**
- Хранение больших объёмов данных.
- Поддержка уровней хранения: Hot, Cool, Archive.

---

### Disk Storage
**Что это:**
- Постоянные управляемые диски для виртуальных машин.

**Особенности:**
- Высокая производительность для рабочих нагрузок.
- Поддержка различных типов дисков (Standard HDD, Standard SSD, Premium SSD).

---

### File Storage
**Что это:**
- Полностью управляемый файловый ресурс с использованием протокола SMB.

**Особенности:**
- Подходит для приложений, требующих совместного доступа к файловым данным.

---

### Storage Tiers (Hot, Cool, Archive)
**Что это:**
- Разные уровни хранения данных в зависимости от частоты доступа к ним.

**Особенности:**
- Hot — часто используемые данные.
- Cool — данные с редким доступом.
- Archive — данные для длительного хранения.

---

## Database services

### Azure SQL Database
**Что это:**
- Управляемая реляционная база данных как сервис (PaaS).

**Особенности:**
- Автоматическое обновление, резервное копирование и масштабирование.
- Высокая доступность.

---

### Azure Database for PostgreSQL
**Что это:**
- Управляемая облачная база данных для PostgreSQL.

**Особенности:**
- Полная совместимость с PostgreSQL.
- Автоматическое масштабирование и резервное копирование.

---

### Azure Database for MySQL
**Что это:**
- Управляемая облачная база данных для MySQL.

**Особенности:**
- Поддержка стандартных инструментов MySQL.
- Высокая доступность и безопасность.

---

### Cosmos DB
**Что это:**
- Глобально распределенная база данных для неструктурированных данных.

**Особенности:**
- Поддержка нескольких моделей данных (ключ-значение, документы, графы).
- Мгновенное масштабирование и низкая задержка по всему миру.

---

## Краткий вывод

> Azure предоставляет широкий спектр сервисов для вычислений, сетей, хранения данных и баз данных, что позволяет строить масштабируемые, отказоустойчивые и высокопроизводительные решения.

# AZ-900: Chapter 2.3 - Describe solutions available on Azure

## Internet of Things (IoT)

### IoT Hub
**Что это:**
- Центральный облачный сервис для подключения, мониторинга и управления миллионами устройств IoT.

**Особенности:**
- Двунаправленная связь между устройствами и облаком.
- Поддержка аутентификации и безопасности устройств.

**Применение:**
- Удалённое управление датчиками, машинами и оборудованием.

---

### IoT Central
**Что это:**
- Управляемая платформа для быстрого создания и настройки IoT-приложений без необходимости глубокого программирования.

**Особенности:**
- Быстрая разработка IoT решений.
- Встроенные шаблоны устройств и мониторинг.

**Применение:**
- Бизнес-решения для отслеживания состояния устройств без необходимости строить сложную инфраструктуру.

---

## Big Data and Analytics

### Azure Synapse Analytics
**Что это:**
- Платформа для интеграции аналитики больших данных и традиционных хранилищ данных.

**Особенности:**
- Выполнение запросов на структурированных и неструктурированных данных.
- Глубокая интеграция с машинным обучением и Power BI.

**Применение:**
- Построение сложной аналитики данных для бизнес-отчетов.

---

### HDInsight
**Что это:**
- Облачная версия популярных open-source платформ для работы с большими данными (Hadoop, Spark, Hive).

**Особенности:**
- Масштабируемые кластеры для обработки больших объёмов данных.

**Применение:**
- Анализ больших объемов логов и событий.

---

### Azure Databricks
**Что это:**
- Интегрированная платформа для обработки больших данных и машинного обучения на основе Apache Spark.

**Особенности:**
- Поддержка коллаборации между разработчиками, аналитиками и специалистами по данным.

**Применение:**
- Построение моделей машинного обучения и аналитических пайплайнов.

---

## Artificial Intelligence (AI) and Machine Learning (ML)

### Azure Machine Learning
**Что это:**
- Платформа для создания, обучения и развертывания моделей машинного обучения.

**Особенности:**
- Поддержка AutoML и инструментов для визуального проектирования моделей.

**Применение:**
- Построение предиктивных моделей для бизнеса.

---

### Cognitive Services
**Что это:**
- Набор готовых API для внедрения возможностей искусственного интеллекта в приложения.

**Особенности:**
- Распознавание речи, изображений, текста.

**Применение:**
- Улучшение пользовательского опыта через умные функции.

---

## Serverless Computing

### Logic Apps
**Что это:**
- Облачный конструктор для автоматизации рабочих процессов без написания кода.

**Особенности:**
- Интеграция с сотнями сервисов (например, Outlook, SharePoint, SQL Server).

**Применение:**
- Автоматизация бизнес-процессов (например, отправка уведомлений, обработка заказов).

---

### Event Grid
**Что это:**
- Сервис маршрутизации событий между источниками и получателями.

**Особенности:**
- Высокая скорость доставки событий.
- Масштабирование в зависимости от нагрузки.

**Применение:**
- Реализация событийных архитектур.

---

### Service Bus
**Что это:**
- Надёжная система обмена сообщениями для интеграции между приложениями и сервисами.

**Особенности:**
- Поддержка очередей и тем (topics).
- Гарантированная доставка сообщений.

**Применение:**
- Построение распределённых систем с асинхронной коммуникацией.

---

# AZ-900: Chapter 2.4 - Describe Azure management tools

## Azure Portal

**Что это:**
- Веб-интерфейс для управления всеми сервисами Azure.

**Особенности:**
- Визуальное управление ресурсами.
- Поддержка настройки, мониторинга и развертывания сервисов.
- Возможность работы с дашбордами для персонализированного отображения информации.

**Применение:**
- Создание, настройка и мониторинг виртуальных машин, сетей, баз данных и других ресурсов.

---

## Azure PowerShell

**Что это:**
- Набор командлетов для автоматизации управления ресурсами Azure через PowerShell.

**Особенности:**
- Полная автоматизация задач администрирования.
- Возможность написания скриптов для комплексных операций.

**Применение:**
- Массовое создание ресурсов.
- Настройка политик безопасности и автоматизация обновлений.

---

## Azure CLI

**Что это:**
- Кросс-платформенная командная строка для управления ресурсами Azure.

**Особенности:**
- Доступна на Windows, macOS и Linux.
- Интуитивно понятные команды.

**Применение:**
- Быстрая настройка сервисов без использования портала.
- Автоматизация процессов через Bash-скрипты.

---

## Azure Cloud Shell

**Что это:**
- Интерактивная браузерная оболочка с предустановленными инструментами Azure CLI и Azure PowerShell.

**Особенности:**
- Работает прямо из Azure Portal.
- Поддержка управления через облако без локальной установки инструментов.

**Применение:**
- Управление ресурсами в любых условиях через веб-браузер.

---

## Azure Advisor

**Что это:**
- Персонализированный консультант по оптимизации ресурсов Azure.

**Особенности:**
- Даёт рекомендации по повышению производительности, безопасности, надежности и оптимизации затрат.

**Применение:**
- Получение советов по улучшению конфигурации и снижению расходов.

---

## Azure ARM templates

**Что это:**
- Декларативные JSON-шаблоны для автоматизированного развертывания ресурсов Azure.

**Особенности:**
- Определяют инфраструктуру как код (Infrastructure as Code).
- Повторное использование и управление версиями инфраструктуры.

**Применение:**
- Автоматизация создания целых сред (виртуальные машины, сети, базы данных) по одному файлу.

---

## Azure Monitor

**Что это:**
- Центральная система для сбора, анализа и визуализации телеметрических данных ресурсов Azure и локальных сред.

**Особенности:**
- Настройка алертов (оповещений).
- Интеграция с Log Analytics и Application Insights.

**Применение:**
- Мониторинг состояния и производительности приложений и инфраструктуры.

---

## Azure Service Health

**Что это:**
- Сервис для информирования о событиях, влияющих на доступность или производительность сервисов Azure.

**Особенности:**
- Предоставление информации о плановых технических работах, сбоях и рекомендации по устранению проблем.

**Применение:**
- Быстрая реакция на проблемы с облачной инфраструктурой.

---

## Краткий вывод

> Azure предоставляет мощные инструменты управления и мониторинга, которые позволяют пользователям легко развертывать, контролировать, оптимизировать и защищать свои облачные ресурсы.
